DOKUMENTASI IMPLEMENTASI SISTEM
Sistem Rekomendasi Playlist Musik Berbasis Konteks Aktivitas Pengguna Menggunakan Metode EDAS

Dokumen ini disusun untuk mendukung penulisan Bab 4, khususnya Subbab 4.3 (Implementasi Sistem).
Format dokumentasi dibuat bertahap per level pengembangan agar konsisten antara praktik coding dan narasi penelitian.

====================================================================
RINGKASAN PENDEKATAN IMPLEMENTASI
====================================================================
Implementasi dilakukan secara bertahap sesuai arsitektur sistem pada paper:
1) Level 1: Foundation Data (preprocessing dan validasi data siap EDAS)
2) Level 2: Konversi Preferensi Pengguna menjadi Bobot + Jenis Kriteria
3) Level 3: Implementasi Perhitungan EDAS (AV, PDA, NDA, SP, SN, NSP, NSN, AS)
4) Level 4: Pembentukan Playlist berdasarkan ranking dan target durasi
5) Level 5: Pembungkusan ke backend API
6) Level 6: Integrasi NLG
7) Level 7: Integrasi frontend

Catatan metodologis penting:
- Variabel aktivitas berfungsi sebagai konteks framing pengisian kuesioner.
- Faktor yang masuk langsung ke perhitungan EDAS adalah hasil jawaban kuesioner
  yang telah dikonversi menjadi bobot kriteria dan jenis kriteria (benefit/cost).

====================================================================
LEVEL 1 - FOUNDATION DATA (SELESAI)
====================================================================
Tujuan Level 1:
Menyiapkan dataset agar valid sebagai matriks keputusan EDAS sesuai rancangan di paper
(Subbab 4.2.5), yaitu memiliki alternatif lagu, 8 kriteria numerik, dan data durasi
untuk pembentukan playlist.

Ekspektasi hasil Level 1:
- Dataset dapat dibaca dari file CSV.
- Data difilter berdasarkan status valid (status = ok).
- Delapan parameter audio tersedia dan bertipe numerik:
  bpm, energy, danceability, happiness, popularity, acousticness,
  instrumentalness, speechiness.
- Durasi lagu berhasil dikonversi dari format menit:detik ke detik.
- Decision matrix terbentuk dengan ukuran (m x 8) tanpa nilai NaN.

--------------------------------------------------
B. Penjelasan fungsi per file
--------------------------------------------------
1) File: backend/src/preprocess.py
   Fungsi utama yang dibuat:

   a. EDAS_CRITERIA
      - Konstanta daftar 8 kriteria EDAS sesuai paper:
        ["bpm", "energy", "danceability", "happiness", "popularity",
         "acousticness", "instrumentalness", "speechiness"]

   b. parse_duration_to_seconds(duration_text)
      - Mengubah format durasi "mm:ss" menjadi detik.
      - Menangani normalisasi ketika ditemukan anomali scraping, misalnya "2:60".

   c. _safe_parse_duration_to_seconds(duration_text)
      - Wrapper aman agar baris dengan format durasi bermasalah tidak membuat
        proses berhenti total.

   d. _ensure_columns(df, required_columns)
      - Memastikan kolom wajib tersedia sebelum preprocessing dilakukan.

   e. preprocess_for_edas(df)
      - Menjalankan pipeline preprocessing:
        1. validasi kolom wajib,
        2. normalisasi/filtrasi kolom status menjadi "ok",
        3. konversi duration -> duration_seconds,
        4. konversi 8 kriteria menjadi numerik,
        5. drop baris invalid,
        6. bentuk decision matrix.
      - Return:
        - cleaned_df (data siap pemrosesan lanjutan)
        - decision_matrix (matriks keputusan EDAS)

2) File: backend/src/main.py
   - Menambahkan pemanggilan preprocess_for_edas(df).
   - Menampilkan indikator validasi Level 1:
     - jumlah baris mentah,
     - jumlah baris setelah filter status,
     - shape decision matrix,
     - jumlah NaN,
  - rentang durasi detik,
  - contoh data siap EDAS.

D. Kendala teknis dan penanganan
--------------------------------------------------
Kendala:
- Terdapat anomali data durasi dengan format "2:60".

Dampak:
- Parsing durasi gagal jika hanya menerima detik 0-59 secara ketat.

Penanganan:
- Parser durasi dinormalisasi sehingga detik >= 60 dikonversi
  ke menit + sisa detik (contoh 2:60 -> 180 detik).
- Baris yang benar-benar tidak valid ditangani aman melalui safe parser.

Kesimpulan kendala:
- Pipeline preprocessing menjadi robust terhadap variasi data scraping,
  tanpa mengubah prinsip perhitungan EDAS.

LEVEL 2 - KONVERSI PREFERENSI KUESIONER (SELESAI)
====================================================================
Tujuan Level 2:
Mengimplementasikan mekanisme konversi jawaban kuesioner (14 butir skala Likert)
menjadi dua komponen utama yang dibutuhkan metode EDAS, yaitu:
1) bobot kriteria, dan
2) jenis kriteria (benefit/cost/neutral).

Ekspektasi hasil Level 2:
- Sistem menerima jawaban 14 butir (skala 1-5) secara lengkap.
- Jawaban dipetakan ke 8 parameter audio sesuai rancangan pada paper.
- Skor parameter dihitung dengan rata-rata per parameter.
- Bobot dinormalisasi sehingga total bobot = 1.
- Jenis kriteria ditentukan dari aturan paper:
  skor > 3 = benefit, skor < 3 = cost, skor = 3 = neutral.

Tujuan Level 3:

  AV, PDA, NDA, SP, SN, NSP, NSN, Appraisal Score.

B. Penjelasan fungsi per file

1) File: backend/src/edas.py
  Komponen utama:

  a. EdasResult
    - Struktur hasil untuk menyimpan seluruh komponen perhitungan:
      average_solution, pda, nda, sp, sn, nsp, nsn, appraisal_score, ranking.

  b. run_edas(decision_matrix, weights, criteria_types)
    - Validasi kolom terhadap bobot dan jenis kriteria.

    - Hitung Appraisal Score dengan rata-rata NSP dan NSN.
    - Hitung ranking menurun berdasarkan Appraisal Score.

  Catatan implementasi untuk kriteria netral:
  - Kriteria berlabel "neutral" diproses dengan arah benefit (tanpa membalik arah),
    sejalan dengan prinsip paper bahwa preferensi netral tidak diperlakukan ekstrem.

  if criterion_type == "cost":
     pda_column = (av - values).clip(lower=0)
     nda_column = (values - av).clip(lower=0)

     nda_column = (av - values).clip(lower=0)

  Penjelasan bahasa manusia:

  - Untuk kriteria cost, arah perhitungan dibalik sesuai rumus EDAS.

3) Cuplikan dari backend/src/edas.py

  sp = weighted_pda.sum(axis=1)
  sn = weighted_nda.sum(axis=1)

  max_sp = float(sp.max())
  max_sn = float(sn.max())

  nsp = sp / max_sp
  nsn = 1 - (sn / max_sn)

  appraisal_score = 0.5 * (nsp + nsn)

  Penjelasan bahasa manusia:
  - Nilai positif dan negatif dijumlahkan berbobot, lalu dinormalisasi.
  - Skor akhir didapat dari gabungan dua sisi tersebut agar penilaian seimbang.

Level 4 konsisten dengan paper pada bagian pembentukan playlist:
- playlist dibentuk dari hasil pemeringkatan EDAS,
- lagu dipilih berurutan dari prioritas tertinggi,
- total durasi disesuaikan dengan input pengguna,
- total akhir dapat mendekati atau sedikit melebihi target durasi.

Status Level 4: Lanjut (siap masuk Level 5 - Pembungkusan Backend API).
